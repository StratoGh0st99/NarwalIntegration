<context>
## Why archived
The Narwal robot broadcasts raw AI detection candidates via display_map field 9/12 during cleaning.
These are every object the camera tentatively identifies — not the confirmed set shown in the app.
The app shows ~5 confirmed objects while the raw stream produces 20-30+ detections with drifting
positions. The confirmed/filtered objects appear to only exist in the app layer (possibly cloud-processed
or onboard post-processing not exposed via the local WebSocket API).

## What we learned about the protocol
- Field 9: sliding window of current detections {1: type_id, 2: seq, 3-5: metadata}
- Field 12: 2 trail segments (field3=1,2 = dual mop tracks) with coords + detection lists
- Both segments contain identical detection lists
- Coordinates are float32 arrays (bbp returns raw bytes, not hex dicts)
- Detection positions = last trail coordinate, which moves with robot
- type_id enum: 1=Cable, 4=Floor Mirror (others unverified, different from APK enum)
